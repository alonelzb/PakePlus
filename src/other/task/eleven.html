1 ansible 常用指令总结，并附有相关示例。 ​

  • /usr/bin/ansible 主程序，临时命令执行工具
  • /usr/bin/ansible-doc 查看配置文档，模块功能查看工具,相当于man
  • /usr/bin/ansible-playbook 定制自动化任务，编排剧本工具,相当于脚本
  • /usr/bin/ansible-pull 远程执行命令的工具
  • /usr/bin/ansible-vault 文件加密工具
  • /usr/bin/ansible-console 基于Console界面与用户交互的执行工具
  • /usr/bin/ansible-galaxy 下载/上传优秀代码或Roles模块的官网平台

利用ansible实现管理的主要方式：

Ansible 即利用ansible命令，主要用于临时命令使用场景

Ansible playbook 主要用于长期规划好的，大型项目的场景，需要有前期的规划过程

ansible 使用前准备

ansible 相关工具大多数是通过ssh协议，实现对远程主机的配置管理、应用部署、任务执行等功能

建议：使用此工具前，先配置ansible主控端能基于密钥认证的方式联系各个被管理节点

bash
# Ping 所有主机
ansible all -m ping

# 执行 Shell 命令
ansible all -a "echo 'Hello, World!'"

# 安装软件包
ansible all -m apt -a "name=nginx state=present"   # 适用于 Ubuntu/Debian
ansible all -m yum -a "name=nginx state=present"   # 适用于 CentOS/RHEL

2. 总结ansible playbook目录结构及文件用途。 ​

Playbook 是一种以 YAML 格式编写的文件，用于定义一组任务的执行顺序和配置。一个 Ansible Playbook 包含一个或多个 Play，而每个 Play 包含一个或多个 Task。

group_vars/ 和 host_vars/： 存放用于定义组变量（group_vars）和主机变量（host_vars）的目录。这些变量可以在 Playbook 中使用，用于配置不同主机或主机组的行为。

roles/： 存放角色的目录，每个角色通常包含 tasks/（执行任务的主要目录）、handlers/（处理任务事件的目录）、files/（存放静态文件的目录）、templates/（存放模板文件的目录） 和 vars/（存放角色变量的目录）等子目录。

playbook.yml： 主 Playbook 文件，定义了一组 Plays 和它们的执行顺序。这是 Ansible Playbook 的入口文件。

inventory.ini： 存放主机清单文件，包含了 Ansible 将要管理的主机信息。该文件定义了主机组、主机别名、主机变量等。

ansible.cfg： 存放 Ansible 配置的文件，包含了 Ansible 运行时的配置选项，如日志路径、模块路径等
