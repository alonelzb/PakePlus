Clash.Meta ​

简介¶ ​

Clash.Meta 是一个基于开源项目 Clash 的二次开发版本，增加了一些独有特性，本文档仍在修订中，欢迎 PR。

使用 Clash.Meta Kernel 前，请悉知：

本 Wiki 基于最新的 Meta Alpha 分支介绍，如果您的 Meta 低于此版本，可能会有部分功能不支持； Meta 核心支持所有原开源核心的全部特性，支持原 Clash Premium 核心部分特性。

Clash.Meta 主要特性 ​

代理模块：¶

  1. 支持出站传输协议 VLESS Reality, Vision 流控
  2. 支持出站传输协议 Trojan XTLS
  3. 支持出站传输协议 Hysteria
  4. 支持出站传输协议 TUIC
  5. 支持出站传输协议 ShadowTLS
  6. 支持 PASS ( 跳过 ) 规则
  7. 主动健康检测 urltest/fallback（基于 tcp 握手，限定时间内多次失败会主动触发健康检测使用节点）
  8. 支持策略组正则筛选
  9. 允许 Provider 请求通过代理更新
  10. Proxy-Providers 支持直接使用 V2rayN 等客户端的普通订阅
  11. Relay 代理链支持 UDP over TCP
  12. TCP 连接并发

规则模块¶ ​

  1. 支持规则 GEOSITE
  2. 支持入站类型规则 IN-TYPE
  3. 支持规则集 RULE-SET
  4. 支持规则 SRC-PORT 和 DST-PORT 的多端口条件
  5. 支持规则对 TCP / UDP 分别管控
  6. 支持 Network 规则，支持匹配网络类型 ( TCP / UDP )
  7. 支持逻辑判断规则 ( NOT / OR / AND )
  8. 支持子规则集
  9. 支持所有规则的源 IPCIDR 条件，只需附加到末尾即可
  10. 支持 GEODATA MODE 切换，mmdb / dat
  11. 支持切换 GEODATA LOADER 模式切换 , 普通 / 小内存模式
  12. 支持 GeoSite 按需加载
  13. 支持使用 geox-url 自定义的 GEOIP / GEOSITE 数据库下载地址

DNS 模块¶ ​

  1. 支持 Sniffer 域名嗅探器
  2. 支持 Fallback-Filter 使用 Geosite
  3. 恢复 Redir-Host 远程解析
  4. 支持使用代理解析 ip
  5. 支持使用 Policy 分流 DNS
  6. 支持 DNS over HTTP/3
  7. 支持 DNS over QUIC

TUN 模块¶ ​

  1. 支持 macOS、Linux 和 Windows
  2. 内置 iptables, 无需手动配置
  3. 内置 Wintun 驱动程序
  4. 支持 gvisor / system / lwip 堆
