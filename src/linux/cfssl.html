CFSSL ​

TIP

CFSSL是CloudFlare开源的一款PKI/TLS工具。 CFSSL包含一个命令行工具和一个用于签名，验证并且捆绑TLS证书的 HTTP API服务。使用Go语言编写。

github: https://github.com/cloudflare/cfssl

CFSSL包括：

  • 一组用于生成自定义 TLS PKI 的工具
  • cfssl程序，是CFSSL的命令行工具
  • multirootca程序是可以使用多个签名密钥的证书颁发机构服务器
  • mkbundle程序用于构建证书池
  • cfssljson程序，从cfssl和multirootca程序获取JSON输出，并将证书，密钥，CSR和bundle写入磁盘

Linux下载 ​

bash
#!/bin/bash

version=1.6.5
url=https://github.com/cloudflare/cfssl/releases/download

# cfssl
xh -d -o cfssl $url/v${version}/cfssl_${version}_linux_amd64
# cfssljson
xh -d -o cfssljson $url/v${version}/cfssljson_${version}_linux_amd64
# cert-info
xh -d -o cfssl-certinfo $url/v${version}/cfssl-certinfo_${version}_linux_amd64

chmod +x cfssl*
mv cfssl* /usr/local/bin

创建根CA证书及其私钥 ​

根CA证书及其私钥，只需要创建一次即可。后续其他证书都由它签名，CA根证书及其私钥一旦改变，其它证书也就无效了。

根证书一般命名是ca.pem

根证书私钥一般命名是ca-key.pem

1 创建根CA证书签名请求文件ca-csr.json ​

ca-csr.json中csr含义是：Certificate Signing Request，证书签名请求

xx-csr.json，该文件是证书签名请求文件，有了它，cfssl就可以生成证书和证书私钥了

csr示例:

也可用 cfssl print-defaults csr 生成默认的csr请求文件进行修改

bash
{
    "CN": "example.com",
    "key": {
        "algo": "rsa",
        "size": 2048
    },
    "names": [
        {
            "C":  "CN",
            "L":  "ChengDu",
            "O":  "test",
            "OU": "dev",
            "ST": "sc"
        }
    ]
}

# 说明
- C: Country， 国家
- L: Locality，地区，城市
- ST: State，州，省
- O: Organization Name，组织名称，公司名称
- OU: Organization Unit Name，组织单位名称，公司部门
- CN: Common Name，浏览器使用该字段验证网站是否合法，一般写的是域名。非常重要。浏览器使用该字段验证网站是否合法

2 生成自签名根 CA 证书和私钥 ​

bash
cfssl genkey -initca ca-csr.json | cfssljson -bare ca
1

这会生成 ca.pem（CA 证书）和 ca-key.pem（CA 私钥）

根据CA根证书及其私钥签发其它证书及其私钥 ​

1 配置证书生成策略，让CA软件知道颁发有什么功能的证书 ​

bash
#打印config模板文件从而进行修改
cfssl print-defaults config > ca-config.json

{
    "signing": {  # 定义签名相关的配置和策略
        "default": {  # 配置会应用于没有特别指定 profile 的证书
            "expiry": "168h"  # 证书的有效期
        },
        "profiles": {  # 允许定义多个签名配置文件（profiles），每个配置文件可以有不同的设置，用于不同类型的证书签发
            "test": {  # # 可自定义
                "expiry": "8760h",
                "usages": [  # 定义证书的用途
                    "signing",  # 允许证书用于签名其他证书
                    "key encipherment",  # 允许证书用于加密密钥,通常是安全传输（如 TLS 握手过程中）所需的
                    "server auth"  # 表示证书可用于服务器身份验证
                    # "client auth"
                ]
            },
            "client": {  # 可自定义
                "expiry": "8760h",
                "usages": [
                    "key encipherment",
                    "client auth"  # 表示证书可用于客户端身份验证
                ]
            }
        }
    }
}

3 使用 CA 签发证书 ​

bash
cfssl gencert -ca=ca.pem -ca-key=ca-key.pem -config=ca-config.json -profile=test test-csr.json | cfssljson -bare server

# -profile=XXX 需要与ca-config.json 中 profiles中的字段对应
# test-csr.json 上面构建的证书签名请求文件
