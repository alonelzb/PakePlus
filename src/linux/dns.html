DNS（Domain Name System） ​

DNS 服务器 ​

域名解析：DNS 服务器将用户提供的域名翻译为与之关联的 IP 地址。这是 DNS 的主要功能之一。

IP 地址分配：DNS 服务器可以用于分配动态 IP 地址给设备，这在局域网中特别有用。这称为 DHCP（Dynamic Host Configuration Protocol）。

缓存：DNS 服务器可以缓存已解析的域名和 IP 地址，以提高查询性能。这有助于减少互联网上的DNS流量。

区域管理：DNS 服务器允许域名的所有者（通常是网站管理员）管理其域名的DNS记录，包括A记录（IPv4地址）、AAAA记录（IPv6地址）、CNAME记录（别名）、MX记录（邮件交换）等。

反向解析：DNS 服务器允许查找IP地址对应的域名，这称为反向解析。反向DNS记录通常用于安全审计和电子邮件验证。

安全功能：某些DNS服务器可以实施安全功能，如DNSSEC（DNS安全扩展）来防止DNS劫持和欺诈。

域名的层级 ​

比如，域名math.stackexchange.com显示为math.stackexchange.com.。这不是疏忽，而是所有域名的尾部，实际上都有一个根域名。

举例来说，www.example.com真正的域名是www.example.com.root，简写为www.example.com.。因为，根域名.root对于所有域名都是一样的，所以平时是省略的。

根域名的下一级，叫做"顶级域名"（top-level domain，缩写为TLD），比如.com、.net；再下一级叫做"次级域名"（second-level domain，缩写为SLD），比如www.example.com里面的.example，这一级域名是用户可以注册的；再下一级是主机名（host），比如www.example.com里面的www，又称为"三级域名"，这是用户在自己的域里面为服务器分配的名称，是用户可以任意分配的。

总结一下，域名的层级结构如下。

bash
主机名.次级域名.顶级域名.根域名

# 即

host.sld.tld.root

资源记录 ​

记录类型:

  • SOA 起始授权记录，一个区域解析库只能有一个SOA，必须位于解析库的第一条记录
  • A
  • AAAA ipv6
  • PTR
  • NS
  • CNAME
  • MX 邮件
  • TXT

区域数据文件 ​

文件位置: /var/named/example.com.zone
格式：name [TTL] IN rr_type value

bash
$TTL 1D
@   IN SOA @ email.com(v1 1D 10M 1D 1H)
        NS 

www     A 1.1.1.1

# 示例 master aaa.email.com(版本号 刷新时间 重试 过期时间 错误记录的缓存时间)
TTL 可从全局继承
使用 @ 引用当前区域的域名
SOA记录的value 说明
两部分内容，中间用空格隔开，
前半部分：Zone的主 DNS 服务器的主机名
后半部分：管理Zone的负责人的电子邮件地址。在该电子邮件名称中使用英文句点 (.) 代替“at”符号 (@)。
serial：数值Serial代表这个Zone的序列号，序号越大表明资料越新,主要用在 master / slave 同步使用，每次Zone文件更新，都需要修改Serial数值。RFC1912 2.2建议的格式为YYYYMMDDnn 其中nn为修订号，,但序号的最大值不能大于2的32次方,即4294967296
refresh：数值Refresh设置Slave DNS多长时间与Master Server进行Serial核对。目前Bind的notify参数可设置每次Master DNS更新都会主动通知Slave DNS更新，Refresh参数主要用于notify参数关闭时;
retry：数值Retry设置当Slave DNS试图获取Master DNS Serial时，如果Master DNS未响应，多长时间重新进行检查;
expire：如果master/slave在1周内都链接失败,slave将不再尝试链接master，但注意，此时slave还会继续提供解析服务，只是数据不在更新。
Minimum：在 8.2版本之前，由于没有独立的 $TTL 指令，所以通过 SOA 最后一个字段来实现。但由于 BIND 8.2 后出现了 $TTL  指令，该部分功能就不再由 SOA 的最后一个字段来负责，由 $TTL 全权负责，SOA 的最后一个字段专门负责 negative answer ttl（negative caching）

在区域数据文件中，没有使用点号"."结尾的，在实际使用的时候都会自动补上域名
例如www 会自动补全为www.example.com

DNS配置的一些文件

  • /etc/hosts　　主机的一个文件列表
    于简单的主机名解析（点分表示法），默认在请求DNS或NIS网络域名服务器前，/etc/named.conf 通常会告诉程序先查看此文件
  • /etc/resolv.conf　　转换程序配置文件
    在配置程序请求BIND域名查询服务查询主机名时，必须告诉程序使用哪个域名服务器和IP地址来完成这个任务
  • /etc/named.conf　　BIND主文件
    设置一般的name参数，指向该服务器使用的域数据库的信息源
  • /var/named/named.ca　　根域名配置服务器指向文件
  • /var/named/localhost.zone　 localhost区正向域名解析文件
    用于将本地IP地址（127.0.0.1）转换为本地回送IP地址（127.0.0.1）
  • /var/named/name.localhost　　localhost区反向域名解析文件
    用于将localhost名字转换为本地回送IP地址（127.0.0.1）
  • /etc/named.rfc1912.zones　　区块设置文件

DNS服务搭建 ​

解析局域网内的自定义域名

在配置Bind服务时，主要用到以下三个配置文件

  1. 主配置文件（/etc/named.conf）
  2. 区域配置文件（/etc/named.rfc1912.zones）
  3. 数据配置文件目录（/var/named）

安装脚本

bash
dns install -y bind bind-utils
domain=asdfgh.com

# 1 更改dns配置文件
sed -i 's/localhost;/any;/' /etc/named.conf
sed -i 's/127.0.0.1/any/' /etc/named.conf
# 配置检查
# named-checkconf

# 2 创建区域记录文件
# touch /var/named/alonelzb.com.zone

cat << EOF > /var/named/${domain}.zone
\$TTL 1D
@  IN SOA master gmail.com(1 1D 10M 1D 1H)
      NS master

        A 106.12.133.68
master  A 1.1.1.1
www     A 106.12.133.68
pic     A 106.12.133.68

# *       A 106.12.133.68
# “*”代表所有，就是泛域名解析
EOF

# 更改权限和组
chmod 640 /var/named/${domain}.zone
chgrp named /var/named/${domain}.zone

# 3 加入配置
cat << EOF >> /etc/named.rfc1912.zones
zone "${domain}" IN {
    type master;
    file "${domain}.zone";
};
EOF

named-checkzone $domain /var/named/${domain}.zone

rndc reload
# systemctl restart named
